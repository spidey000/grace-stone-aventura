# Style Guide — Aventura tipo Jumanji

## Objetivo

La web debe sentirse como una expedición misteriosa, táctil y cinematográfica. La referencia principal es una aventura selvática tipo Jumanji, protagonizada por el carismático Dr. Smolder Bravestone: más exploración, más reliquias, más mapa antiguo y menos interfaz genérica.

La experiencia visual debe transmitir:
- aventura
- misterio
- tesoro oculto
- peligro controlado
- descubrimiento progresivo

La sensación general no debe ser plana ni tecnológica. Debe parecer un juego narrativo con piezas coleccionables, pistas, guardianes y un mapa vivo.

## Principios Visuales

1. **Mundo físico, no digital**
   - Todo debe parecer hecho de materiales reales: madera, piedra, pergamino, metal envejecido, tela, cristal, hojas, cuerda.
   - Evitar superficies limpias y minimalistas.
   - Las capas visuales deben sugerir profundidad, desgaste y uso.

2. **Misterio accesible**
   - El diseño debe ser oscuro y atmosférico, pero legible y amable para niños.
   - El contraste debe permitir lectura clara sin perder la sensación de selva nocturna.

3. **Narrativa visible**
   - Cada pantalla debe parecer un capítulo de una expedición.
   - Los componentes deben parecer parte del mundo: mapa, reliquia, bússola, insignias, fragmentos, guardián.

4. **Progreso como recompensa**
   - El usuario debe sentir avance físico: piezas que encajan, fragmentos que se iluminan, objetos que se desbloquean.
   - Las animaciones deben celebrar logros, no distraer.

## Dirección Estética

### Referencias de ambiente
- selva húmeda
- ruinas antiguas
- templo escondido
- brújula y cartografía
- runas y símbolos tallados
- luz cálida de linterna
- polvo en el aire
- hojas en sombra

### Sensaciones a priorizar
- tensión suave
- curiosidad
- recompensa
- descubrimiento
- aventura familiar

## Paleta de Color

La base debe apoyarse en tonos orgánicos y contrastes profundos.

### Colores principales
- `#1f3a2e` verde jungla oscuro
- `#2f5d50` verde musgo
- `#6b4f2a` marrón madera
- `#8a6a3d` cuero envejecido
- `#c9a15a` oro antiguo
- `#f2d8a7` pergamino
- `#101613` sombra casi negra

### Colores de acento
- `#d96b3b` naranja antorcha
- `#c73e3e` rojo reliquia
- `#4f93d2` azul cristal
- `#6fd1c6` turquesa tesoro

### Uso recomendado
- fondo general: degradados oscuros verdosos y marrones
- botones primarios: oro antiguo o antorcha
- estados correctos: verde hoja o turquesa luminosa
- advertencias o tensión: rojo reliquia, con moderación
- paneles narrativos: pergamino, madera o piedra clara

## Fondos Y Capas

Los fondos deben sentirse profundos y vivos.

### Fondo base
- degradado radial u oblicuo de verde oscuro a negro
- textura sutil de selva, niebla o grano
- sombras difusas en bordes para enmarcar el contenido

### Capas decorativas
- hojas grandes en opacidad baja
- brillos de luz filtrada entre árboles
- polvo o partículas flotantes
- siluetas de ruinas o tótems
- mapas y rutas apenas visibles

### Regla de densidad
- el fondo puede ser rico, pero el contenido principal siempre debe ganar
- no usar decoraciones de alto contraste detrás de texto o controles

## Tipografía

La tipografía debe reforzar el tono de expedición.

### Jerarquía sugerida
- títulos: serif con presencia, estilo explorador o editorial
- subtítulos: serif o sans condensada con personalidad
- cuerpo: sans legible, cálida y moderna
- microtexto: sans simple, siempre muy clara

### Sensación tipográfica
- títulos con autoridad de mapa antiguo
- texto narrativo con tono de cuento o leyenda
- botones con peso visual suficiente para parecer objetos interactivos

### Reglas
- no usar tipografías impersonales de UI como estilo dominante
- evitar exceso de mayúsculas
- no sacrificar legibilidad por ambientación

## Formas Y Componentes

### Tarjetas
- forma de pergamino, placa o reliquia
- esquinas ligeramente redondeadas, no demasiado suaves
- borde visible, con textura o relieve
- sombra cálida, no gris fría

### Botones
- deben parecer piezas sólidas
- usar relieve, borde y cambio de estado claro
- el hover debe sentirse como “descubrir” o “activar”

### Insignias y chips
- pequeñas medallas, amuletos o fragmentos
- colores por estado, con bordes metálicos o tallados

### Banners narrativos
- parecen carteles antiguos o cintas de expedición
- pueden incluir iconos de brújula, mapa, antorcha o pieza de rompecabezas

### Progreso
- barras, fragmentos o contadores deben parecer artefactos
- preferir piezas que se iluminan o encajan antes que barras planas

## Iconografía

La iconografía debe mezclar aventura y misterio.

### Temas de iconos
- brújula
- mapa
- fragmento
- llave
- antorcha
- piedra rúnica
- ojo antiguo
- huella
- pluma
- gema
- cuerda

### Estilo
- siluetas claras
- trazo grueso o forma sólida
- alto reconocimiento a tamaños pequeños
- si se usa emoji temporalmente, debe sentirse consistente y no infantil

## Ilustración Y Objetos

Los objetos visuales deben parecer hallazgos del viaje.

### Materiales
- madera tallada
- piedra erosionada
- metal oxidado
- vidrio o cristal brillante
- pergamino envejecido
- telas tipo lona o mapa

### Reglas
- usar brillos con moderación
- evitar superficies plásticas
- los objetos coleccionables deben distinguirse con claridad del entorno

## Motion

El movimiento debe parecer exploración y descubrimiento, no UI genérica.

### Animaciones recomendadas
- pulso lento en objetos activos
- aparición con escala suave para fragmentos encontrados
- brillo breve cuando una pieza encaja
- desplazamientos cortos y con peso
- partículas o luces muy sutiles en fondos

### Ritmo
- animaciones suaves, con sensación orgánica
- evitar movimientos rápidos o hiperactivos
- cada animación debe tener función narrativa

### Estados
- activo: una señal clara de energía o latido
- completado: pequeño estallido de recompensa
- bloqueado: quieto, pesado, casi sellado

## UI Por Sección

### Lobby
- debe parecer la entrada a la expedición
- protagonista visual: título grande, objeto central y llamada a la acción fuerte
- fondo con selva, mapa o ruina

### Pantalla de acertijo
- sensación de cámara de explorador o panel de misión
- banner narrativo superior
- pasos como fragmentos o placas
- feedback visible y celebratorio

### Pantalla final
- debe sentirse como la cámara del tesoro
- más luz, más oro, más brillo
- revelado final con sensación de triunfo

### Elementos de ayuda
- deben parecer guías o herramientas de expedición
- usar tono amistoso, nunca burocrático

## Tono De Ilustración Y Copy

El lenguaje visual y textual deben trabajar juntos.

### Tono
- curioso
- épico pero cercano
- ligeramente enigmático
- familiar y seguro

### Evitar
- tono corporativo
- tono demasiado infantil
- humor moderno que rompa la inmersión
- referencias tecnológicas obvias

## Accesibilidad

La atmósfera no debe perjudicar la usabilidad.

- contraste suficiente para texto y botones
- no depender solo del color para estados
- animaciones suaves y no invasivas
- evitar fondos recargados detrás de contenido crítico
- iconos y ornamentos siempre secundarios al contenido

## Reglas De Uso

### Sí
- usar paletas orgánicas y profundas
- hacer que cada pantalla parezca parte de una expedición
- tratar componentes como objetos físicos
- reforzar progreso, tesoro y exploración

### No
- usar estética plana de dashboard
- usar blancos fríos dominantes
- usar gradientes genéricos sin textura
- usar sombras blandas de SaaS moderno
- usar iconografía demasiado limpia o corporativa

## Frase De Control

Si una pantalla no parece una pieza de una aventura en la selva, no está alineada con este estilo.

## Aplicación A Esta Web

La web del Dr. Smolder Bravestone debe combinar:
- la aventura selva/tesoro como base visual
- los elementos de fragmentos, mapas y guardianes ya definidos en `ai-assets`
- una presentación cálida, inmersiva y narrativa

La meta es que el usuario sienta que está resolviendo una expedición antigua, no simplemente navegando una web.
